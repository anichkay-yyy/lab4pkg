# Спецификация проекта

## Диаграммы

Все диаграммы находятся в папке `diagrams/` в формате PlantUML (.puml).

### Диаграмма вариантов использования
![Use Case](../diagrams/use_case.puml)

Файл: `diagrams/use_case.puml`

Описание: Отображает двух актёров (Администратор и Маклер) и их взаимодействие с системой. Администратор имеет полный доступ ко всем функциям. Маклер может просматривать свои сделки, добавлять сделки и просматривать отчеты.

### Диаграммы деятельности

**Аутентификация** — `diagrams/activity_login.puml`

Описание: Процесс входа в систему: ввод логина/пароля, хэширование, проверка в БД, маршрутизация по роли.

**CRUD Сделки** — `diagrams/activity_deal.puml`

Описание: Процесс управления сделками: добавление с автообновлением статистики, редактирование, удаление, просмотр.

### Диаграммы классов
Файл: `diagrams/class.puml`

Описание: Структура модулей приложения. Основные классы: Database, Auth, User, Broker, Goods, Deal, Queries, Functions, BrokerStats. Показаны связи между модулями и таблицами БД.

### Диаграммы последовательности

**Аутентификация** — `diagrams/sequence_login.puml`

Описание: Взаимодействие между пользователем, main.c, auth.c и SQLite при входе в систему.

**Добавление сделки** — `diagrams/sequence_deal.puml`

Описание: Полный цикл добавления сделки: ввод данных, INSERT в PB_DEALS, автоматическое обновление PB_BROKER_STATS.

### Диаграмма компонентов
Файл: `diagrams/component.puml`

Описание: Общая структура проекта: исходные файлы, заголовки, тесты, БД. Показаны зависимости между модулями.

### Диаграмма развертывания
Файл: `diagrams/deployment.puml`

Описание: Схема развертывания: компьютер пользователя (приложение + SQLite), GitHub (два репозитория, GitHub Actions, Wiki, Pages), Codecov.
