# Функциональные требования

## Варианты использования

### Актёры
- **Администратор** — полный доступ ко всем функциям
- **Маклер** — доступ только к своим данным (пункты, помеченные *)

### Функциональные требования

#### FR-1. Аутентификация
Система должна обеспечивать вход пользователя по логину и паролю с определением роли (администратор или маклер).

#### FR-2. Управление маклерами (только администратор)
- FR-2.1. Просмотр списка всех маклеров
- FR-2.2. Добавление нового маклера (фамилия, адрес, год рождения)
- FR-2.3. Редактирование данных маклера
- FR-2.4. Удаление маклера

#### FR-3. Управление товарами (только администратор)
- FR-3.1. Просмотр списка всех товаров
- FR-3.2. Добавление нового товара (название, вид, цена, поставщик, срок годности, количество)
- FR-3.3. Редактирование данных товара
- FR-3.4. Удаление товара

#### FR-4. Управление сделками
- FR-4.1. Просмотр списка сделок (маклер видит только свои)
- FR-4.2. Добавление новой сделки с автоматическим обновлением статистики
- FR-4.3. Редактирование сделки (только администратор)
- FR-4.4. Удаление сделки (только администратор)

#### FR-5. Аналитические отчеты
- FR-5.1. По названию товара — проданное количество и общая стоимость за период
- FR-5.2. По названию товара — перечень фирм-покупателей с количеством и стоимостью
- FR-5.3. По виду товара с наибольшим спросом — количество и стоимость по покупателям (*)
- FR-5.4. По маклеру с максимальным количеством сделок — сведения о маклере и фирмах-поставщиках
- FR-5.5. По каждой фирме-поставщику — список маклеров с количеством и стоимостью

#### FR-6. Специальные функции
- FR-6.1. Обновление статистики маклера при совершении сделки (*)
- FR-6.2. Консолидация товаров на указанную дату (вычитание проданного, удаление сделок)
- FR-6.3. Вывод всей информации о сделках маклера на указанную дату (*)

### Текстовый сценарий: «Добавление сделки»

**Основной поток:**
1. Маклер выбирает пункт меню «Добавить сделку»
2. Система запрашивает данные: дату сделки, название товара, вид, количество, фирму-покупателя
3. Маклер вводит данные
4. Система проверяет существование маклера в БД (FK constraint)
5. Система добавляет запись в таблицу PB_DEALS
6. Система автоматически обновляет статистику маклера в PB_BROKER_STATS
7. Система выводит подтверждение: «Сделка добавлена (ID: N)»

**Альтернативный поток 4a:** Указан несуществующий маклер — система выводит ошибку «FOREIGN KEY constraint failed»

**Альтернативный поток 5a:** Количество <= 0 — система выводит ошибку «CHECK constraint failed»
